У status_parser есть два параметра:
    --offset [число] - пропускает указанное количество записей
    --table [0 или 1] - 1 (стандартное значение) - записи берутся уже из готовй базы
                      - 0 -  то качает файл и читает данные из него


1. Парсер статусов "get_statuses.py" спарсит всё примерно за 5000000/(10000/300)/60/60 = 42 часа
2. Парсер сайтов "site_parser.py" спарсит всё примерно за ...

Тест (пустая база) №1: Парсинг 70876 сайтов закончился за 2190 секунд
Тест (сразу после первого теста) №2: Парсинг 70876 сайтов закончился за 2131 секунд
